## E2E Flow Verification

### Flow 1: New User Signup ✓

1. User visits `/signup`
2. Sees ToS checkbox (links to `/terms`)
3. Sees no-PHI checkbox
4. Must check both to submit (Zod validation)
5. Submits → Supabase auth signup
6. First login → layout checks `terms_acknowledged_at`
7. Modal shows (non-dismissible)
8. User acknowledges → API updates profile
9. User sees app with persistent banner

**Status:** COMPLETE

1. User logs in
2. Layout queries `terms_acknowledged_at`

**Status:** COMPLETE
