**Modify `app/app/api/transcribe/route.ts`:**

The transcription route currently uses a direct `fetch()` to the OpenAI Whisper API. Unlike generate/template routes, we cannot simply swap to `getModel()` because transcription is not a language model -- it requires different APIs per provider. For Phase 31, we add config awareness while keeping the OpenAI implementation intact. Phase 33 will add Groq Whisper as an alternative.

1. **Add** import after the existing imports (after line 12):
   ```typescript
   import { getTranscriptionConfig } from '@/lib/ai/config';
   ```

2. **Remove** the OPENAI_API_KEY check block (lines 237-248). Replace with a provider-aware check. Delete:
   ```typescript
   // Check for OpenAI API key
   if (!process.env.OPENAI_API_KEY) {
     console.error('OPENAI_API_KEY is not configured');
     return new Response(
       JSON.stringify({
         success: false,
         error: 'Configuration Error',
         message: 'AI transcription service is not configured. Please contact support.',
       }),
       { status: 500, headers: { 'Content-Type': 'application/json' } }
     );
   }
   ```
   
   Replace with:
   ```typescript
   // Get transcription provider configuration from environment
   const transcriptionConfig = getTranscriptionConfig();
   
   // Validate required API key for the configured provider
   if (transcriptionConfig.provider === 'openai' && !process.env.OPENAI_API_KEY) {
     console.error('[Transcribe] OPENAI_API_KEY is not configured for OpenAI transcription');
     return new Response(
       JSON.stringify({
         success: false,
         error: 'Configuration Error',
         message: 'AI transcription service is not configured. Please contact support.',
       }),
       { status: 500, headers: { 'Content-Type': 'application/json' } }
     );
   }
   ```

3. **Add** a comment before the Whisper API call block (before line 304) to mark the provider-specific section:
   ```typescript
   // Provider: OpenAI Whisper (default)
   // Future: Phase 33 will add Groq Whisper v3 Turbo branch based on transcriptionConfig.provider
   ```

4. **Do NOT change** the actual OpenAI Whisper fetch logic, FormData construction, or response handling. The transcription implementation stays identical -- we are only adding the config plumbing so Phase 33 can branch on `transcriptionConfig.provider`.

This is a minimal change: add the import, swap the API key check to be provider-aware, and add a comment marking where the provider branching will go. The actual Whisper API call, retry logic, usage tracking, and response format are untouched.
