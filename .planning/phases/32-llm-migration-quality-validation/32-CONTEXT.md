# Phase 32: LLM Migration & Quality Validation - Context

**Gathered:** 2026-04-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Radiology reports are generated by Groq Llama 4 Scout at ~96% lower cost with quality validated against GPT-4o baseline. This phase adapts the system prompt for open-source models, switches the model via env vars (using Phase 31 abstraction), and creates a quality validation suite of 25 test cases across modalities.

</domain>

<decisions>
## Implementation Decisions

### Prompt Adaptation Strategy
- Move examples from system prompt to user prompt to get under 2K tokens; keep rules in system prompt
- Convert "NEVER/CRITICAL" anti-hallucination directives to numbered constraints with explicit pass/fail criteria (Groq migration guide recommendation)
- Start temperature at 0.2 (matching current GPT-4o setting) for medical determinism
- Keep adapted prompt in same file (generate/route.ts) with clear comments, colocated with usage

### Quality Validation Approach
- Test cases stored as JSON fixtures in `app/lib/ai/test-fixtures/` with input findings + expected GPT-4o baseline output
- Automated structural checks (section presence, heading format) + manual review flag for hallucination detection
- 25 test cases: 5 MRI brain, 5 CT chest, 5 X-ray chest, 5 ultrasound abdomen, 5 mixed/edge cases
- Pass criteria: 100% structure compliance, 100% anti-hallucination, >95% template adherence

### Model Integration
- Use `llama-4-scout-17b-16e-instruct` via `@ai-sdk/groq` provider
- Set env vars: `AI_PROVIDER=groq`, `AI_MODEL=llama-4-scout-17b-16e-instruct` — Phase 31 registry resolves
- Switch all text generation routes (generate, templates/generate, templates/suggest) to Groq together
- `GROQ_API_KEY` env var validated at startup by registry

### Claude's Discretion
- Exact prompt restructuring approach (which examples move, which rules consolidate)
- Test fixture content (specific medical findings and expected outputs)
- Validation script implementation details

</decisions>
