Per CONTEXT.md decisions on prompt adaptation strategy:

1. **Restructure system prompt to under 2K tokens.** The current prompt is ~2.5K tokens across ~120 lines. Apply these specific changes:

   a. **Move examples from system prompt to user prompt.** The 4 EXAMPLE blocks (CORRECT, INCORRECT HALLUCINATION, CORRECT NORMAL, INCORRECT HALLUCINATION -- lines ~257-270) are ~200 tokens. Move them into the user prompt section, prefixed with "REFERENCE EXAMPLES:" before the findings.

   b. **Convert CRITICAL/NEVER directives to numbered constraints with pass/fail criteria.** Per Groq migration guide recommendation, open models respond better to explicit numbered rules than emphatic directives. Transform the anti-hallucination rules into:
      ```
      CONSTRAINTS (violations cause report rejection):
      1. ONLY report findings explicitly stated in USER FINDINGS. Pass: every abnormality traces to input. Fail: any added finding.
      2. NEVER add measurements not in input. Pass: all numbers match input. Fail: any invented measurement.
      3. NEVER elaborate beyond what was dictated. Pass: abnormality descriptions match input detail level. Fail: added specifics.
      4. Normal findings MAY be added ONLY for structures NOT mentioned in input.
      5. No contradictions within same organ system. Pass: consistent characterization. Fail: "normal" and "abnormal" for same structure.
      6. No "Pertinent Negatives" section or similar headings.
      ```

   c. **Consolidate redundant instructions.** The current prompt repeats the anti-hallucination concept in 3 places (CRITICAL RULES, REPORTING STANDARDS, NORMAL FINDINGS INTEGRATION). Merge into the single CONSTRAINTS block.

   d. **Add explicit reasoning chain instruction for Llama models:**
      ```
      REASONING PROCESS (follow this order):
      Step 1: Read all user findings and list the mentioned structures.
      Step 2: For each mentioned structure, report ONLY what was stated.
      Step 3: For unmentioned structures, add template normal findings.
      Step 4: Check for contradictions between findings.
      Step 5: Write Impression summarizing key findings only.
      ```

   e. **Keep these unchanged:** Section structure (## headings), bold subsection format, Impression bullet format, temperature 0.2.

2. **Update the user prompt** to include the moved examples before the findings text. Structure:
   ```
   REFERENCE EXAMPLES (for formatting guidance):
   [moved examples here]

   USER FINDINGS / DICTATION:
   [findings]

   ANTI-HALLUCINATION CHECKLIST:
   [simplified checklist]

   Generate a professional radiology report in Markdown format now.
   ```

3. **Verify token count.** After restructuring, count approximate tokens (rough: words * 1.3). System prompt must be under 2K tokens (~1500 words max). If still over, further trim the OUTPUT FORMAT section by removing the detailed example and keeping just the section structure specification.

4. Keep `temperature: 0.2` (per CONTEXT.md: start at 0.2 matching current GPT-4o setting).

5. Keep `maxOutputTokens: 2000` unchanged.

IMPORTANT: The adapted prompt must preserve ALL safety rules -- just restructured, not weakened. Every anti-hallucination rule must remain. The goal is format optimization for open models, not content reduction.
