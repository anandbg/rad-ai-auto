<objective>
Complete Phase 34 by (1) adding Groq → OpenAI fallback to the transcription route and switching to real duration-based cost tracking, and (2) lowering the daily cost ceiling default from $20 to $5 to match the 71x cheaper Groq baseline while still protecting against the ~30x fallback amplification.

Purpose: Research flagged that the current $20/day ceiling is 71x too generous for Groq-baseline traffic — it would never trigger even during sustained abuse. But it must also not be so low that a legitimate fallback-to-OpenAI day (30x cost spike) causes false emergencies. $5 is the sweet spot: it caps Groq-baseline at ~7,000 reports/day while tripping into warning mode around ~250 OpenAI-fallback reports/day (clear operator signal).

Output: Updated transcribe route with fallback + real-duration cost, updated ceiling module with new default, and unit tests for the ceiling boundaries.
</objective>

       ```typescript
       async function callWhisperAPI(
         provider: string,
         model: string,
         audioFile: File
       ): Promise<{ text: string; duration?: number }> {
         const fd = new FormData();
         fd.append('file', audioFile, audioFile.name);
         fd.append('prompt', RADIOLOGY_VOCABULARY_HINT);
         fd.append('model', model);
         fd.append('response_format', 'verbose_json');

         const { apiUrl, apiKey } = provider === 'groq'
           ? { apiUrl: 'https://api.groq.com/openai/v1/audio/transcriptions', apiKey: process.env.GROQ_API_KEY! }
           : { apiUrl: 'https://api.openai.com/v1/audio/transcriptions',      apiKey: process.env.OPENAI_API_KEY! };

         const response = await fetch(apiUrl, {
           method: 'POST',
           headers: { 'Authorization': `Bearer ${apiKey}` },
           body: fd,
         });

         if (!response.ok) {
           const errorData = await response.json().catch(() => ({}));
           const error = new Error(errorData?.error?.message || `${provider} transcription error: ${response.status}`);
           (error as Error & { status: number }).status = response.status;
           throw error;
         }

         return response.json();
       }
       ```

    3. Replace the existing `withRetry(...)` block with a fallback-aware version:

       ```typescript
       const primary = getTranscriptionConfig(); // { provider, model }
       const FALLBACK = { provider: 'openai', model: 'whisper-1' };

       let servedBy: { provider: string; model: string };
       let result: { text: string; duration?: number };

       try {
         result = await withRetry(
           () => callWhisperAPI(primary.provider, primary.model, audioFile),
           { maxRetries: 3, operationName: `transcribe:${primary.provider}` }
         );
         servedBy = primary;
       } catch (primaryError) {
         // Skip fallback if primary IS openai (would be double-tap)
         if (primary.provider === 'openai') {
           throw primaryError;
         }
         console.warn(
           `[Transcribe] Primary ${primary.provider} failed, falling back to openai:whisper-1:`,
           primaryError instanceof Error ? primaryError.message : primaryError
         );
         try {
           result = await withRetry(
             () => callWhisperAPI(FALLBACK.provider, FALLBACK.model, audioFile),
             { maxRetries: 2, operationName: 'transcribe:openai-fallback' }
           );
           servedBy = FALLBACK;
         } catch (fallbackError) {
           console.error('[Transcribe] Both providers failed', { primaryError, fallbackError });
           throw fallbackError;
         }
       }
       ```
